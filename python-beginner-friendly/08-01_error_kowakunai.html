Chapter 08

エラーを怖がらない

エラーメッセージは問題を教えてくれる親切な案内人です

エラーは敵じゃない、味方です

プログラミングでは、エラーは必ず出ます。プロのプログラマーでも毎日エラーに出会います。大事なのは「エラーが出ないこと」ではなく「エラーの意味を読み取ること」です。

安心してください
エラーを出しても、パソコンが壊れることはありません。何度でもやり直せます。

エラーメッセージの読み方

エラーメッセージは英語ですが、見るべきポイントは2つだけです。

---> 3 print("hello"
           ^
SyntaxError: incomplete input
見るべき2つのポイント
1. 最後の行 = エラーの種類と理由
2. 矢印が指す行番号 = 問題がある場所

Excelで言うと、セルにエラーが出たとき「#VALUE!」や「#REF!」と表示されるのと同じ仕組みです。Pythonのエラーメッセージはもっと親切に理由まで教えてくれます。

よくあるエラー 1: SyntaxError

SyntaxErrorは「書き方の間違い」です。第4章のif文でインデントを忘れたときにも出ましたね。

# カッコの閉じ忘れ
print("こんにちは"
SyntaxError: incomplete input
直し方
カッコ、クォーテーション、コロンの閉じ忘れがないか確認しましょう。対になる記号を数えるのがコツです。
やってみよう

ヒント

よくあるエラー 2: NameError

NameErrorは「その名前、知りません」というエラーです。変数名のタイプミスが原因のことがほとんどです。

name = "田中"
print(nama)  # nameのつもりが nama
NameError: name 'nama' is not defined
直し方
エラーメッセージに「name 'nama'」と書いてあります。「nama」という変数は定義されていないよ、という意味です。スペルを確認しましょう。

よくあるエラー 3: TypeError

TypeErrorは「データの型が合っていない」というエラーです。第3章で文字列と数値を足そうとしたときに出ましたね。

age = 25
print("年齢は" + age)  # 文字列+数値はNG
TypeError: can only concatenate str to str
直し方
f文字列を使いましょう: f"年齢は{age}" または str(age) で文字列に変換します。
やってみよう

3つのエラー早見表

Syntax
SyntaxError
書き方の間違い。カッコ、コロン、クォーテーションの閉じ忘れ、インデントのミスなど。
Name
NameError
変数名のスペルミスや、定義していない変数を使おうとした場合。
Type
TypeError
データの型が合わない操作。文字列と数値を直接足すなど。
CHECKPOINT

ここまでのポイント

  • エラーはパソコンが壊れるわけではない、何度でもやり直せる
  • エラーメッセージは「最後の行」と「行番号」を見る
  • SyntaxError = 書き方のミス、NameError = 名前のミス、TypeError = 型のミス
  • エラーが出ても慌てずにメッセージを読めば、必ず直せる

まとめ: エラーを怖がらない

  • エラーは味方: 問題の場所と原因を教えてくれる親切なメッセージです
  • 読み方は2つ: エラーの種類（最後の行）と場所（行番号）を確認する
  • 3大エラー: SyntaxError、NameError、TypeErrorの対処法を覚えれば大丈夫
