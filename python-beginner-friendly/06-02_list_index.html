「3番目のデータだけ取り出したい」を実現しよう

前回の復習

前のトピックでリストの作り方を学びました。
今回は、リストの中の特定のデータを取り出す方法を学びます。

names = ["田中", "鈴木", "佐藤", "高橋"]
# この中の「鈴木」だけ取り出すには？
Excelで言うと

A列に名前が並んでいて、「A2セルの値を見る」ような操作です。
Pythonでは「番号」を使ってリストの特定の要素にアクセスします。

インデックス = 番号

インデックスは0から始まる！これが最初の注意ポイント

0番目から始まる！

最初の要素は 0番目。ここが一番の注意点です。

names = ["田中", "鈴木", "佐藤", "高橋"]
print(names[0])
print(names[1])
田中
鈴木
0から始まる理由

日常生活では「1番目」から数えますが、プログラミングでは「0番目」から始まります。
最初は混乱しますが、すぐに慣れますのでご安心ください。

後ろから数える：マイナスのインデックス

最後の要素を取り出したいときは [-1] が便利です。

names = ["田中", "鈴木", "佐藤", "高橋"]
print(names[-1])
print(names[-2])
高橋
佐藤
使いどころ

[-1] は「最後の要素」を意味します。リストの長さを気にせず、常に最後のデータを取り出せるので、とても便利です。

やってみよう

インデックスを使ってみよう

チャレンジ：特定のメンバーを取り出そう

Google Colabで以下を入力して実行してみましょう。

team = ["田中", "鈴木", "佐藤", "高橋"]
print(f"最初: {team[0]}")
print(f"最後: {team[-1]}")
できましたか？

[0] で最初の要素、[-1] で最後の要素が取り出せます。
[2] にすると何が表示されるか予想してみてください！

範囲外アクセスのエラー体験

存在しない番号を指定するとエラーになります。
これも大切な学びなので、実際に試してみましょう。

names = ["田中", "鈴木", "佐藤"]
print(names[5])  # 5番目は存在しない！
IndexError: list index out of range
IndexError とは

「リストの範囲外」というエラーです。3個しかない（インデックス0, 1, 2）のに5番目を見ようとした、ということです。
慌てずに、インデックスの数字を確認しましょう。

やってみよう

エラーを体験してみよう

チャレンジ：わざとエラーを出してみよう

エラーは怖くありません。わざと出して慣れましょう！

colors = ["赤", "青", "緑"]
print(colors[10])  # わざとエラー
大丈夫！

IndexErrorが出ましたね。このエラーを見たら「あ、番号が間違っている」とすぐわかるようになります。
エラーは敵ではなく、味方です！

業務での活用例

月別の売上リストから、特定の月のデータを取り出す

sales = [500000, 620000, 480000]
print(f"1月: {sales[0]}円")
print(f"最新月: {sales[-1]}円")
1月: 500000円
最新月: 480000円
実務のポイント

売上データや顧客リストなど、順番に並んだデータから特定の位置のデータを取り出す場面は多いです。[-1] で「最新のデータ」を取り出すパターンは特によく使います。

ここまでのポイント

インデックスの基本を確認

  • インデックスは 0 から始まる（1ではない）
  • [-1] で最後の要素にアクセスできる
  • 範囲外を指定すると IndexError が出る
  • エラーが出たら番号を確認すればOK

まとめ：インデックスでアクセス

  • リスト名[番号] で要素を取り出す 0番目が最初、1番目が2つ目。0から始まるのがポイント。
  • マイナスのインデックス [-1]で最後、[-2]で最後から2番目。リストの長さを気にせず使える。
  • IndexError は範囲外のサイン エラーが出たら「番号が大きすぎないか？」を確認しよう。
