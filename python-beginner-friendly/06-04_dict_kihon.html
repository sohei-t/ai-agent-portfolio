辞書の基本
名前でデータを管理

ExcelのVLOOKUPのように
「名前で検索してデータを取り出す」

リストの不便な点

リストは番号でアクセスしました。
でも「3番目の人」ではなく「田中さんのデータ」と指定したくないですか？

# リストだと...
scores = [85, 92, 78]
# scores[0] が誰の点数？ わからない！
Excelで言うと

A列に名前、B列に点数が入っている表を想像してください。
VLOOKUP関数で「田中」を検索して点数を取り出す。辞書はそれと同じです。

辞書 = キーと値のペア

辞書はキーで値を引く「電話帳」のようなもの

辞書の作り方

波かっこ { } で囲み、キー: 値 のペアをカンマで区切ります。

scores = {
    "田中": 85,
    "鈴木": 92,
    "佐藤": 78
}
リストとの違い

リストは [ ] 角かっこ、辞書は { } 波かっこ。
リストは番号でアクセス、辞書は名前（キー）でアクセスします。

キーで値を取り出す

scores = {"田中": 85, "鈴木": 92}
print(scores["田中"])
print(scores["鈴木"])
85
92

「田中さんの点数は？」→ scores["田中"] で85が返る。
番号ではなく名前で取り出せるので直感的ですね。

やってみよう

辞書を作ってみよう

チャレンジ：社員の部署情報を辞書で管理
department = {
    "田中": "営業部",
    "鈴木": "開発部",
    "佐藤": "総務部"
}
print(department["鈴木"])
開発部
できましたか？

名前を指定するだけで部署がわかります。
自分の会社のメンバーに変えて試してみましょう！

新しいキーの追加

辞書にはいつでも新しいデータを追加できます。

scores = {"田中": 85, "鈴木": 92}
scores["佐藤"] = 78
print(scores)
{'田中': 85, '鈴木': 92, '佐藤': 78}
値の更新も同じ書き方

既に存在するキーに代入すると値が更新されます。
scores["田中"] = 90 で田中さんの点数が90に変わります。

やってみよう

辞書にデータを追加しよう

チャレンジ：辞書を更新してみよう
prices = {"りんご": 150}
prices["バナナ"] = 100
prices["みかん"] = 80
print(prices)
{'りんご': 150, 'バナナ': 100, 'みかん': 80}

辞書の基本を確認

  • 辞書は { } で作り、キー: 値 のペアを入れる
  • 辞書["キー"] で値を取り出す
  • 辞書["新キー"] = 値 で追加、既存キーなら更新
  • リストは「番号」、辞書は「名前」でアクセス

まとめ：辞書の基本

  • 辞書 = キーと値のペア 波かっこで囲み、コロンでキーと値をつなぐ。VLOOKUPのようなもの。
  • 名前でアクセスできる 「田中さんの点数」のように直感的に取り出せる。
  • 追加も更新も簡単 辞書["キー"] = 値 で、新規追加も値の更新もできる。
