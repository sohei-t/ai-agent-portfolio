# ピアノ演奏アプリ - 要件定義書（最終版）

**バージョン**: 1.0 Final
**更新日**: 2024-12-09
**改善イテレーション**: 3回完了

## 1. プロジェクト概要

### 1.1 プロジェクト名
**スマホピアノ (Smartphone Piano App)**

### 1.2 目的
スマートフォンやPCブラウザで簡単にピアノ演奏を楽しめるWebアプリケーションを開発する。初心者でも直感的に操作でき、直感的なUIで音楽を楽しめる環境を提供する。

### 1.3 ターゲットユーザー
- ピアノ初心者（楽器を持っていない人）
- 音楽に興味がある一般ユーザー
- 移動中や待ち時間に音楽を楽しみたい人
- 子供から大人まで幅広い年齢層

## 2. 機能要件

### 2.1 必須機能（MVP Phase 1 - 3日間）

#### 2.1.1 ピアノ鍵盤表示
- **鍵盤数（明確化）**：
  - スマホ縦向き: 1オクターブ = **12鍵**（C4-B4）
  - スマホ横向き（〜375px幅）: 1.5オクターブ = **18鍵**（C4-F5）
  - スマホ横向き（376-768px幅）: 2オクターブ = **24鍵**（C4-B5）
  - タブレット/PC: 2.5オクターブ = **30鍵**（C4-F6）
- **鍵盤デザイン**：
  - 白鍵と黒鍵を視覚的に区別
  - タップ/クリック時の視覚的フィードバック（押下効果、色変化）
  - **黒鍵タップ領域拡大**（視覚的幅より大きく、最小44x44px）
- **レスポンシブ対応**：
  - 画面サイズに応じた鍵盤サイズ自動調整
  - タッチデバイス最適化（`touch-action: manipulation`）

#### 2.1.2 音源生成
- **技術**: Web Audio API
- **音質**: シンセサイザー音源（Oscillator、無料、外部ファイル不要）
- **仕様**:
  - 各鍵盤に対応する周波数で音を生成（A4=440Hz基準）
  - タップ/クリック時に即座に音が鳴る（目標レイテンシ < 30ms）
  - 音の減衰（ADSR エンベロープ）を実装
  - 同時発音対応（最大10音、古いノード自動削除）
  - **iOS Safari対応**（初回ユーザー操作後にAudioContext初期化）
- **初期化設定**:
  ```javascript
  const audioContext = new AudioContext({ latencyHint: 'interactive' });
  ```

#### 2.1.3 オクターブ切り替え
- **範囲**: C3-C7（4オクターブ範囲）
- **UI**: ボタン（▲▼）またはスライダーで切り替え
- **表示**: 現在のオクターブ位置を明示（例: "C4"）

#### 2.1.4 音名表示切り替え
- **表示モード**:
  - 日本語（ド・レ・ミ・ファ・ソ・ラ・シ）
  - 英語（C・D・E・F・G・A・B）
  - 非表示
- **設定保存**: LocalStorageで設定を保持

#### 2.1.5 ビジュアルテーマ切り替え
- **テーマ数**: 3種類
  - **クラシック**（白黒の伝統的ピアノ）
  - **モダン**（グラデーション、影付き）
  - **ネオン**（暗い背景に光る鍵盤、グローエフェクト）
- **適用範囲**: 鍵盤、背景、UI全体
- **切り替え**: ドロップダウンまたはアイコンボタン
- **実装**: CSS Variables + `data-theme` 属性
- **アニメーション**: 0.3秒のスムーズなトランジション

#### 2.1.6 音量調整（Phase 1追加）
- **UI**: スライダー（0-100%）
- **機能**: マスター音量調整
- **ミュート**: ワンクリックミュートボタン

#### 2.1.7 キーボード操作（PC向け、Phase 1追加）
- **対応**: QWERTY配列で鍵盤演奏
- **マッピング例**:
  ```
  Q=C, W=D, E=E, R=F, T=G, Y=A, U=B （白鍵）
  2=C#, 3=D#, 5=F#, 6=G#, 7=A# （黒鍵）
  ```
- **視覚フィードバック**: キーボード押下時も鍵盤が反応

### 2.2 Phase 2機能（2日間）

#### 2.2.1 録音・再生機能
- **録音**:
  - 演奏を記録（noteOn/noteOffイベント形式）
  - 最大録音時間: 5分
  - 録音中の視覚的表示（録音ボタンが赤く点滅）
- **再生**:
  - 録音した演奏を再生
  - 再生中の鍵盤アニメーション表示
  - 一時停止/停止機能
- **保存**:
  - LocalStorageに保存（最大10件、約1MB）
  - JSON形式（詳細は技術仕様参照）

#### 2.2.2 練習曲機能（簡易版）
- **楽曲数**: 1曲（きらきら星のみ）
- **機能**:
  - 自動再生（お手本演奏）
  - テンポ調整（0.5倍速〜2倍速）

#### 2.3.2 メトロノーム機能

## 3. 非機能要件

### 3.1 性能要件（厳格化）
- **レスポンス時間**:
  - 鍵盤タップから音が鳴るまで < **30ms**（目標値を厳格化）
  - テーマ切り替え < 200ms
  - 初期ロード < **2秒**（3Gネットワーク）
- **同時発音数**: 最大10音（古いノード自動削除）
- **FPS**: 60fps維持（アニメーション時）
- **バンドルサイズ**: < 300KB

### 3.2 使いやすさ（UX）
- **直感性**: チュートリアルなしで基本操作が理解できる（30秒以内に演奏開始）
- **視覚的フィードバック**: すべての操作に明確なフィードバック
- **エラーハンドリング**: 音が鳴らない場合の案内表示（iOS Safari対応）

### 3.3 アクセシビリティ（追加）
- **WAI-ARIA**: ボタンに`aria-label`付与
- **キーボードナビゲーション**: Tabキーで移動可能
- **カラーコントラスト**: WCAG AA準拠

### 3.4 互換性
- **対応ブラウザ**:
  - Chrome 最新版（推奨）
  - Safari 最新版（iOS対応重要）
  - Firefox 最新版
  - Edge 最新版
- **対応デバイス**:
  - iOS（iPhone、iPad） - **最優先**
  - Android（スマートフォン、タブレット）
  - PC（Windows、Mac、Linux）

### 3.5 セキュリティ
- **データ保護**: 録音データは端末内のみに保存（外部送信なし）
- **外部依存**: CDNなしで動作（オフライン可能、Viteでバンドル化）

### 3.6 保守性
- **コード品質**:
  - ESLint/Prettier準拠
  - コメント記載（主要関数）
  - モジュール化された構造（コンポーネント単位）
- **テストカバレッジ**: 80%以上

## 7. リスクと対策（詳細化）

### 7.1 リスク一覧

| リスク | 影響度 | 発生確率 | 対策 | 検証方法 |
|--------|--------|----------|------|----------|
| iOS Safari自動再生制限 | 高 | 高 | 初回タップでAudioContext初期化 | iPhone実機テスト |
| 黒鍵のタップ領域が小さい | 中 | 中 | タップ領域拡大（44x44px最小） | スマホ実機テスト |
| Web Audio APIメモリリーク | 中 | 中 | OscillatorNode自動削除（onended） | 長時間演奏テスト |
| LocalStorage容量不足 | 低 | 低 | 容量警告、古いデータ削除 | 10件録音テスト |

### 7.2 主要リスクへの詳細対策

#### 7.2.1 iOS Safari対応（最重要）
```javascript
// 初回タップ時にAudioContext初期化
document.addEventListener('touchstart', () => {
  if (audioContext.state === 'suspended') {
    audioContext.resume();
  }
}, { once: true });
```

#### 7.2.2 黒鍵タップ領域拡大
```css
.black-key {
  width: 30px; /* 視覚的な幅 */
  padding: 10px; /* タップ領域拡大 */
  min-width: 44px; /* iOS HIG準拠 */
  touch-action: manipulation;
}
```

#### 7.2.3 メモリリーク防止
```javascript
osc.onended = () => {
  osc.disconnect();
  gainNode.disconnect();
};
```

## 11. 用語集

- **Web Audio API**: ブラウザでオーディオ処理を行うJavaScript API
- **Oscillator**: 音を生成する波形発振器
- **ADSR**: Attack, Decay, Sustain, Release（音の減衰特性）
- **LocalStorage**: ブラウザにデータを永続化する仕組み（5-10MB）
- **レスポンシブデザイン**: 画面サイズに応じてレイアウトを最適化する設計
- **CSS Variables**: CSS内で変数を定義し、テーマ切り替えに使用
- **Latency**: 入力から出力までの遅延時間
